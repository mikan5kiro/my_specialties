# 拿手菜小程序 - 功能需求文档 V3.2（定稿）

## 产品定位

按使用习惯自动排序的私人菜谱助手。

**核心痛点**：收藏的菜谱太多找不到、常做的菜沉底、边看边做手忙脚乱。

**解决方案**：按制作频率和喜爱度自动排序 + 自由粘贴解析 + 做完打卡自动置顶 + 搜索筛选一站式。

---

## 一、核心功能

### 1. 首页（搜索+排序+筛选 三合一）

搜索框、标签筛选、三Tab排序全部整合在一个页面。

**页面布局**：
- 顶部：搜索框 + 筛选按钮
- 中间：三个Tab（我常做的 | 我爱吃的 | 最近想做）
- 下方：菜谱卡片列表
- 悬浮：＋ 添加按钮

**三Tab排序规则**：

| Tab      | 排序逻辑                       |
| -------- | ------------------------------ |
| 我常做的 | 按制作次数降序                 |
| 我爱吃的 | 按喜爱度评分降序               |
| 最近想做 | 手动标记的菜谱，按标记时间倒序 |

**搜索功能**：
- 输入关键词，从当前Tab数据中筛选
- 搜索范围：菜名 + 食材标签 + 食材清单内容
- 模糊匹配，实时过滤
- 搜索结果保持当前Tab的排序规则

**标签筛选**：
- 三个维度可多选，支持跨维度组合：
  - 食材：鸡肉、牛肉、猪肉、羊肉、虾、鱼、蟹、贝类、鸡蛋、豆腐、菌菇、土豆、番茄、绿叶菜、根茎类
  - 类型：荤菜、素菜、汤羹、主食、凉菜、饮品、烘焙、火锅、烧烤
  - 场景：减脂餐、快手菜、下饭菜、宴客菜、早餐、夜宵、便当、解馋零食
- 标签筛选与关键词搜索可同时生效

**组合示例**：Tab"我常做的" + 搜索"鸡" + 标签"减脂餐" → 常做的鸡肉类减脂菜，按制作次数排列。

---

### 2. 自由粘贴解析

从任意平台复制菜谱文字，粘贴后点击"解析"，系统自动拆分结构化数据。

**解析能力**：

| 提取项 | 识别规则                                     |
| ------ | -------------------------------------------- |
| 菜名   | 首行文字，或【】《》内的文字                 |
| 食材   | 带数量词的行（500g、2勺、少许、适量等）      |
| 步骤   | 包含烹饪动词的行（切、炒、煮、炖、煎、蒸等） |
| 备注   | 小贴士、注意、PS开头的内容                   |
| 过滤   | emoji、#话题、@用户、纯感叹夸奖等            |

**不依赖顺序**：基于关键词匹配（数字+单位 → 食材，烹饪动词 → 步骤），信息顺序打乱也能正确识别。

**解析示例**：

用户粘贴（小红书复制）：
```
🍗新手也能成功的红烧肉！

五花肉500g，冰糖20g，葱姜料酒
生抽2勺、老抽1勺

1️⃣五花肉切块焯水
2️⃣炒糖色
3️⃣加入调料炖40分钟
4️⃣大火收汁

真的巨好吃！！#家常菜 #下饭菜
```

解析结果：
- 菜名：红烧肉
- 食材：五花肉500g、冰糖20g、葱、姜、料酒、生抽2勺、老抽1勺
- 步骤：①五花肉切块焯水 ②炒糖色 ③加入调料炖40分钟 ④大火收汁

**解析后操作**：
- 解析结果自动填入对应字段
- 用户可直接修改、删减、补充
- 确认无误后保存

**切换手动模式**：页面提供切换按钮，手动模式为传统分字段表单。

**技术实现**：微信云函数，预置烹饪动词库 + 规则匹配，无第三方API，免费。

---

### 3. 做完打卡

菜谱详情页底部有醒目的"我又做了"按钮，点击后：
- 制作次数 +1
- 更新"最近制作时间"为当前时间
- 菜谱在"我常做的"列表自动上升
- Toast提示："👍 第N次做这道菜了！"

---

### 4. 喜爱度评分

详情页提供1-5星评分：
- 影响"我爱吃的"Tab排序
- 可随时修改

---

### 5. "最近想做"标记

每道菜可标记为"最近想做"：
- 在"最近想做"Tab中集中展示
- 做完一次后自动取消标记
- 也可手动取消

## 二、菜谱数据结构

```
{
  基本信息：
  - 菜名（必填，文本）
  - 封面图片（可选，云存储文件ID）
  - 食材清单（文本，用户自由输入）
  - 完整步骤（文本，用户自由输入）
  - 备注（可选，文本）
  
  分类标签（多选）：
  - 食材标签
  - 类型标签
  - 场景标签
  
  系统自动维护：
  - 制作次数（整数，初始0）
  - 最近制作时间（时间戳，初始null）
  - 创建时间（时间戳，自动生成）
  
  用户手动操作：
  - 喜爱度（整数1-5，初始0表示未评分）
  - 最近想做（布尔值，初始true）
}
```

## 三、页面结构

总共4个页面：

**首页**
- 搜索框 + 筛选按钮
- 三Tab切换（我常做的 / 我爱吃的 / 最近想做）
- 菜谱卡片列表
- 悬浮添加按钮

**菜谱详情页**
- 封面大图
- 菜名、标签展示
- 喜爱度星级评分
- 制作次数 + 最近制作时间
- 食材清单
- 完整步骤
- 备注
- "我又做了"按钮（突出醒目）
- "最近想做"标记开关
- 编辑按钮 / 删除按钮

**添加/编辑菜谱页**
- 模式切换按钮（粘贴解析 / 手动填写）
- 粘贴模式：大文本输入框 → 解析按钮 → 结果自动填入各字段，用户可修改
- 手动模式：分字段表单
- 图片上传
- 标签选择器
- 保存按钮

**个人中心**
- 总菜谱数
- 总制作次数

---

## 四、操作流程

**找菜做菜**：
打开首页 → 看排序列表或搜索筛选 → 点进详情 → 看步骤做菜 → 点"我又做了"

**添加菜谱**：
点＋ → 粘贴文字 → 点解析 → 检查修改 → 选标签 → 传图 → 保存

**管理菜谱**：
详情页 → 编辑 / 评分 / 标记"最近想做" / 删除

## 六、明确不做的功能

| 功能                            | 原因                         |
| ------------------------------- | ---------------------------- |
| 解析后纠错交互（拖拽/一键移动） | 先让用户手动改，看效果再迭代 |
| 大模型AI解析                    | 规则解析已够用，不增加成本   |
| 图片/视频识别                   | 技术复杂，体验繁琐           |
| 日历视图                        | 与核心痛点关联不大           |
| 游戏化徽章/连续打卡             | 第一版保持简洁               |
| 备菜清单、计时器                | 后期可加                     |
| 多用户/登录系统                 | 单用户私人工具               |
| 社交分享、视频存储              | 不需要                       |
